## 1. 项目概述

### 1.1 项目名称

**旅途博客**（TravelBlog）— 个人博客与旅行生活分享平台

### 1.2 项目目标

构建一个功能完整的个人博客网站，集博客展示、社区交流、风景记录、旅游规划于一体，支持用户注册/登录（含邮箱验证码）、实时互动通知、第三方地图API集成等高阶特性。

### 1.3 用户角色

| 角色 | 描述 |
|------|------|
| 游客（未登录） | 可浏览首页、公开博客、公开相册、旅游建议；不可发帖/评论/点赞 |
| 注册用户 | 可发帖、评论、点赞、创建相册、创建旅游计划；可管理自己的内容 |
| 管理员 | 拥有后台全部权限：用户管理、内容审核、数据统计、系统设置 |

| 缓存/会话 | Redis | 7.x | 验证码、会话、热数据缓存 |
| ORM | TypeORM | 0.3.x | 数据库操作 |
| 实时通信 | Socket.IO Server | 4.x | WebSocket 服务端 |
| 搜索引擎 | Elasticsearch | 8.x（可选）| 全文搜索博客/帖子 |
| 文件存储 | 阿里云 OSS / MinIO | - | 图片、相册文件存储 |
| 邮件服务 | nodemailer + SMTP | - | 发送验证码邮件 |
| 地图服务 | 高德地图 Web API / JS API | - | 路线规划、地理编码 |

### 2.2 技术选型理由

- **NestJS**：模块化架构、装饰器风格、内置 WebSocket 网关与依赖注入，适合中大型全栈项目。
- **PostgreSQL**：支持 JSON 字段、全文检索、ACID 事务，适合复杂关联查询。
- **Redis**：验证码 5 分钟过期、会话 Token 黑名单、排行榜缓存、WebSocket 房间管理。
- **Socket.IO**：成熟可靠的双向通信方案，内置房间与命名空间，适合实时通知场景。
- **Ant Design**：后台管理系统开箱即用，减少 UI 开发成本。

## 4. 功能模块详述

### 4.1 用户模块

#### 4.1.1 注册流程

```
用户填写注册信息（昵称、邮箱、密码）
        │
        ▼
前端校验格式（邮箱格式、密码强度等）
        │
        ▼
POST /api/auth/send-code  ←── 后端生成 6 位随机验证码
        │                         存入 Redis，key: `email_code:{email}`
        │                         TTL: 5 分钟
        ▼
发送验证码邮件（SMTP）
        │
        ▼
用户输入验证码
        │
        ▼
POST /api/auth/register  ←── 后端校验验证码
        │                        从 Redis 获取并比对
        ▼
验证通过 → 创建用户（密码 bcrypt 哈希）→ 返回 JWT Token
```

**密码要求**：至少 8 位，包含大小写字母和数字  
**验证码约束**：60 秒内不允许重复发送（Redis 标记），5 分钟内有效  
**注册限制**：同一 IP 每小时最多注册 3 个账号

#### 4.1.2 登录方式

**方式一：账号密码登录**
```
POST /api/auth/login/password
Body: { email, password }
Response: { accessToken, refreshToken, userInfo }
```

**方式二：邮箱验证码登录**
```
POST /api/auth/login/code
Body: { email, code }
流程：发送验证码 → 验证 → 登录成功 → 返回 Token
```

#### 4.1.3 JWT 令牌策略

| 令牌类型 | 有效期 | 用途 |
|----------|--------|------|
| Access Token | 2 小时 | 请求鉴权 |
| Refresh Token | 7 天 | 续签 Access Token |

#### 4.1.4 用户信息管理

- 获取/修改个人资料（头像、昵称、简介）
- 修改密码（需旧密码验证）
- 密码重置（邮箱验证码流程）

### 4.3 交流社区模块

#### 4.3.1 帖子管理

- **创建帖子**：标题 + 内容（支持 Markdown/富文本）+ 标签 + 可选图片
- **编辑帖子**：仅作者和管理员可编辑
- **删除帖子**：仅作者和管理员可删除（软删除）
- **帖子列表**：分页展示，可按最新/最热/标签筛选
- **帖子详情**：展示全文、标签、发布时间、作者、点赞数、评论列表

#### 4.3.2 点赞功能

- 用户对帖子/评论进行点赞或取消点赞
- 同一用户对同一内容只能点赞一次
- 实时更新点赞计数（Redis 缓存 + 异步落库）

#### 4.3.3 评论功能

- **一级评论**：直接回复帖子
- **二级评论（嵌套）**：回复某条评论（支持@用户）
- **排序**：默认按时间正序
- **分页**：评论列表延迟加载

#### 4.3.4 实时通知（核心特性）

**触发场景**：
1. 有人回复了用户的帖子 → 通知帖子作者
2. 有人回复了用户的评论 → 通知评论作者
3. 有人点赞了用户的帖子/评论 → 通知被点赞者

**技术实现**：

```
用户 A 评论了用户 B 的帖子
        │
        ▼
CommentService.create()
        │
        ├──→ 保存评论到数据库
        │
        └──→ NotificationService.notify()
                 │
                 ├──→ 存入数据库通知表
                 │
                 └──→ Socket.IO emit('notification', payload)
                          │
                          ▼
                  用户 B（在线）← 实时收到通知弹窗
                  用户 B（离线）← 下次登录时拉取未读通知
```

**通知内容**：
```json
{
  "id": "uuid",
  "type": "comment" | "reply" | "like",
  "actor": { "id": "user_id", "nickname": "...", "avatar": "..." },
  "targetType": "post" | "comment",
  "targetId": "uuid",
  "targetTitle": "帖子标题（摘要）",
  "content": "评论/回复内容摘要",
  "isRead": false,
  "createdAt": "ISO8601"
}
```

**前端处理**：
- Socket.IO 客户端连接时携带 JWT Token 进行身份认证
- 加入自己的用户房间 `user:{userId}`
- 收到通知时：顶部弹窗提示 + 通知中心红点计数
- 通知中心页面：列表展示所有通知，支持标记已读

### 4.4 风景记录模块

#### 4.4.1 相册管理

**相册属性**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 名称 | string | 相册标题 |
| 描述 | string | 相册描述 |
| 封面图 | string | 封面图片 URL |
| 照片列表 | string[] | 图片 URL 数组 |
| 可见性 | enum | `private`（默认）/ `public` |
| 标签 | string[] | 自定义标签 |
| 创建时间 | datetime | - |
| 更新时间 | datetime | - |
| 点赞数 | number | 仅公开相册可被点赞 |
| 访问量 | number | - |

#### 4.4.2 可见性控制

- **私有相册**：仅创建者可见，不出现在搜索/推荐中
- **公开相册**：可被其他用户搜索到、查看详情、点赞
- **切换可见性**：用户可随时在编辑相册时切换

#### 4.4.3 相册搜索

- 按相册名称、标签、描述关键词搜索
- 仅搜索公开相册
- 支持分页

#### 4.4.4 点赞排行 & TOP10 轮播

**点赞计数**：Redis ZSET 结构，key: `album:like_rank`，score 为点赞数

```
每次点赞/取消 → Redis ZINCRBY 更新分数
```

**TOP10 获取**：
```
ZREVRANGE album:like_rank 0 9 WITHSCORES
```

**轮播实现**：
- 页面加载时调用 `GET /api/albums/top-liked` 获取 TOP10
- 前端使用轮播组件（Ant Design Carousel / Swiper）展示
- 自动轮播间隔：5 秒
- **用户可选择关闭**：localStorage 存储偏好 `carouselDisabled: true`
- 关闭后显示静态网格布局，点击可重新开启

#### 4.5.4 高德地图 API 调用方案

**所需 API**：

| API 名称 | 用途 | 备注 |
|----------|------|------|
| 地理编码/逆地理编码 | 地址 ↔ 经纬度转换 | 将用户输入的文字地址转为坐标 |
| 路径规划 API（驾车/步行/公交） | 路线规划 | 根据出发地、途经点、目的地规划路线 |
| POI 搜索 API | 周边游玩/餐饮推荐 | 搜索目的地周边的景点、美食 |
| 公共交通 API（高德） | 市内公共交通推荐 | 地铁、公交线路查询 |

**距离判断逻辑**（决定推荐飞机/高铁/火车/公交）：

```
计算出发地与目的地直线距离
│
├── > 800 km → 推荐飞机（同时显示高铁作为备选）
├── 200 ~ 800 km → 推荐高铁（同时显示飞机/火车）
├── 50 ~ 200 km → 推荐火车/高铁 + 当地公共交通
├── 10 ~ 50 km → 推荐公交/地铁/打车
└── < 10 km → 步行/骑行/打车
```

**重要限定**：高德地图 API Key 需用户自行申请，配置在环境变量中。项目中提供配置入口文档。

## 7. 实时通信方案

### 7.1 Socket.IO 架构

```
Socket.IO Server (NestJS Gateway)
         │
         ├── 命名空间: /notifications
         │
         │  连接认证：@WebSocketGateway 中通过 JWT 验证
         │  客户端加入房间: user:{userId}
         │
         │  事件列表：
         │  ┌────────────────────────┬──────────────────────────┐
         │  │ 服务端 → 客户端        │ payload                  │
         │  ├────────────────────────┼──────────────────────────┤
         │  │ notification           │ 新的通知对象              │
         │  │ unread_count           │ { count: number }        │
         │  │ post_updated           │ 帖子更新消息              │
         │  └────────────────────────┴──────────────────────────┘
         │
         │  ┌────────────────────────┬──────────────────────────┐
         │  │ 客户端 → 服务端        │ payload                  │
         │  ├────────────────────────┼──────────────────────────┤
         │  │ mark_read              │ { notificationId }       │
         │  │ mark_all_read          │ {}                       │
         │  │ join                   │ { room: string }         │
         │  └────────────────────────┴──────────────────────────┘
```

### 7.2 实时推送场景

| 场景 | 推送对象 | 推送内容 |
|------|----------|----------|
| 帖子被评论 | 帖子作者 | 评论者昵称 + 评论内容摘要 |
| 评论被回复 | 评论作者 | 回复者昵称 + 回复内容摘要 |
| 帖子/评论被点赞 | 内容作者 | 点赞者昵称 |
| 相册被点赞 | 相册作者 | 点赞者昵称 |

### 7.3 离线消息

- 通知数据同时写入数据库 `notifications` 表
- 用户上线时：Socket.IO 连接成功后，前端调用 `GET /api/notifications/unread-count`
- 用户在线时：实时推送，不重复拉取

| 地理编码 URL | - | `https://restapi.amap.com/v3/geocode/geo` |
| 路径规划 URL | - | `https://restapi.amap.com/v3/direction/driving` |
| POI 搜索 URL | - | `https://restapi.amap.com/v3/place/text` |

**高德 API 调用频率限制**：
- 个人开发者：每日调用量有限，需在代码中做缓存（Redis 缓存热门结果）
- 缓存策略：同一地点的地理编码结果缓存 7 天
- 路径规划结果缓存 1 小时

### 9.5 安全措施

| 类别 | 措施 |
|------|------|
| 密码安全 | bcrypt 哈希，salt rounds = 10 |
| JWT 安全 | Access Token 短期有效，Refresh Token 轮换 |
| 防暴力破解 | 登录失败 5 次后锁定账号 30 分钟（Redis 计数） |
| XSS 防护 | 输入消毒（sanitize-html），输出编码 |
| CSRF 防护 | Token 存储在 HttpOnly Cookie 或 Authorization Header |
| SQL 注入 | TypeORM 参数化查询 |
| 文件上传 | 校验 MIME 类型、文件大小限制、重命名 |
| 速率限制 | 接口级别限流（express-rate-limit / @nestjs/throttler） |
| 请求验证 | class-validator + class-transformer DTO 校验 |
